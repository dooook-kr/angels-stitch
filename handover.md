# 📅 Cstitch 프로젝트 인수인계 문서 (최종 업데이트: 2026-04-26)

이 문서는 AI 에이전트 간의 원활한 프로젝트 인수인계를 위해 작성되었습니다. 앱의 핵심 아키텍처는 Vanilla JS/CSS/HTML 위주의 경량화된 SPA (Single Page Application)이며, 상태 관리는 IndexedDB를 사용합니다.

---

## ✅ 최근 작업 완료 사항 (2026-04-26 세션)

### 1. 🔴 주요 버그 및 크래시 완전 해결 (Critical Issues Fixed)
* **스티치 모드 캔버스 미표시 버그 수정**: `switchTab` 이후 캔버스가 0 사이즈로 렌더링되던 문제를 **이중 `requestAnimationFrame` + 100ms Fallback**을 통해 DOM 레이아웃이 완전히 완료된 후 크기를 측정하도록 수정 완료.
* **초기화 크래시 방지**: `app.js`에서 존재하지 않던 DOM 요소(`stitch-toggle-mode`, `stitch-export-pdf`)를 HTML에 추가하고, 불필요한 바인딩(`stitch-color-list`) 및 존재하지 않는 함수(`debouncedPreview`) 호출을 제거하여 런타임 에러 완전 해결.
* **실 리스트 렌더링 수정**: 상세보기에서 실 이름이 "DMC undefined"로 뜨던 문제를 해결 (`c.dmcId` → `c.code` 매핑 수정).

### 2. 📱 네비게이션 흐름 구조적 개편 (Navigation Refactoring)
* **다이렉트 파일 다이얼로그**: [Create] 탭 또는 홈의 [새 도안 만들기] 버튼을 클릭하면 불필요한 중간 화면 없이 **즉시 시스템의 이미지 선택 팝업이 뜨도록** 로직 개선. 사진 선택 시 즉시 크롭 단계로 전환됨.
* **프로젝트 상세보기 (Project Detail View)**:
  * Project 목록에서 카드를 누르면 바로 스티치 모드로 진입하지 않고, **새롭게 추가된 상세보기 페이지**로 진입.
  * 상세보기에서 썸네일, 통계(격자/색상/진행률), 사용된 실 정보 파악 가능.
  * 상세보기 화면 내에서 PDF 다운로드, 스티치 모드 진입, 삭제 및 이름 저장이 가능하도록 중앙 집중화.
* **스티치 왕복 흐름 개선**: 스티치 모드에서 종료(`X` 버튼) 시 리스트 뷰가 아닌 해당 도안의 상세보기로 되돌아오도록 하여 자연스러운 연속적 작업 흐름(Workflow) 보장.

### 3. 🎨 스티치 모드 UI/UX 레이아웃 안정화
* **캔버스 영역 확보**: 스크롤이 많은 실 리스트가 상단 캔버스를 화면 밖으로 밀어내지 않도록 CSS 레이아웃 전면 수정.
* **고정 비율 폐기**: 기존 `60:40` 비율 대신 캔버스 컨테이너에 `height: 45vh; min-height: 200px`를 부여하여 무조건 화면의 상단 일정 영역을 차지하도록 강제. 하단 실 리스트 패널은 남은 공간을 채우고(`flex: 1`) 내부 독립 스크롤 가능하도록 구현.

## 💡 개발 가이드 (For AI Agent)
* **기존 설계 존중**: 이 앱은 React/Vue 등의 무거운 라이브러리를 배제한 순수 Vanilla 기반입니다. DOM 조작은 `$` (querySelector)와 `$$` 헬퍼를 사용합니다.
* **모바일 퍼스트**: 모든 CSS 레이아웃은 모바일 세로 뷰를 기준으로 우선 설계되었습니다. CSS 수정 시 데스크탑 화면보다 모바일 폭(Width 320~430px)에서의 작동을 최우선으로 고려하세요.
* **상태 관리**: 진행 데이터는 모두 `Storage` 모듈(`IndexedDB`)을 통해 저장됩니다. 상태 모델 수정 시 하위 호환성을 신경쓰세요.
